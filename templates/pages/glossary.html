Security
Glossary

Cybersecurity terms explained in plain language. No jargon, just clarity.

A

API (Application Programming Interface)
A set of rules that allows different software applications to communicate with each other. APIs can be a security risk if not properly protected, as they often handle sensitive data.
Attack Surface
All the points where an attacker could try to enter or extract data from your systems. The larger your attack surface, the more vulnerable you are. Includes websites, APIs, employee devices, and third-party integrations.
Authentication
The process of verifying someone's identity. "Who are you?" Common methods include passwords, biometrics, and security tokens. Different from authorization (what you're allowed to do).

B

Brute Force Attack
An attack that tries every possible combination to guess passwords or encryption keys. Like trying every key on a keyring until one works. Prevented by strong passwords, account lockouts, and rate limiting.
Business Email Compromise (BEC)
A scam where attackers impersonate executives or vendors via email to trick employees into transferring money or sharing sensitive data. One of the most costly cyber threats to businesses.

E

Encryption
Converting data into a coded format that can only be read with the correct key. Like putting a message in a locked box that only the recipient can open. Essential for protecting data in transit and at rest.
Endpoint
Any device that connects to your network: laptops, phones, tablets, servers. Each endpoint is a potential entry point for attackers, which is why endpoint protection is critical.

M

MFA (Multi-Factor Authentication)
Requiring two or more verification methods to access an account: something you know (password), something you have (phone), or something you are (fingerprint). Blocks 99% of automated attacks.
Malware
Short for "malicious software." Any software designed to harm, exploit, or otherwise compromise a computer system. Includes viruses, ransomware, spyware, and trojans.

P

Phishing
Fraudulent emails, texts, or websites designed to trick you into revealing sensitive information or clicking malicious links. The most common attack vector for businesses. Always verify before clicking.
Penetration Testing (Pen Test)
A simulated cyberattack against your systems to check for exploitable vulnerabilities. Ethical hackers try to break in before real attackers do. Should be done regularly by qualified professionals.

R

Ransomware
Malware that encrypts your files and demands payment for the decryption key. A devastating attack that can shut down businesses. Prevention: regular backups, security training, and keeping software updated.

S

Social Engineering
Manipulating people into giving up confidential information or access. Attackers exploit trust, fear, or urgency. Includes phishing, pretexting, and baiting. Your people are often the weakest link.
SQL Injection
An attack where malicious code is inserted into database queries through user input fields. Can allow attackers to read, modify, or delete your database. Part of OWASP Top 10 "Injection" category.

Z

Zero-Day
A vulnerability unknown to the software vendor. Called "zero-day" because developers have had zero days to fix it. Highly valuable to attackers. This is why keeping software updated is crucial the instant patches are released.
Zero Trust
A security model based on "never trust, always verify." No user or device is trusted by default, even inside the network. Access is granted on a need-to-know basis and continuously verified.
